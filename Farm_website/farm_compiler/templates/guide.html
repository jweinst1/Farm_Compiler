Containers

Farm uses two containers of values to compose the output of every program, the alphabet container and the num_sym container. The alphabet container is numerated from 0 to 25, with the pointer initially starting at a, or 0.

The num_sym container is a little different, since it's values are arranged with numbers 0 through 9 first, and the remaining symbols are arranged from the most used to the least used. Here is a table of them:

Patterns

In Farm, it's really simple to make patterns, using loops, and loops of loops. For example, lets say you have want to keep repeating the pattern a3a3. The basic way do this is:

,++.,.,.,.,

a3a3a3a3

However, this can get pretty cumbersome after a while. The more effecient way one can do this is use a series of ! commands.

++,.,.!4,.,.!4

a3a3a3a3a3a3a3a3

There is an even shorter way to accomplish this, with the use of ? loops. Many ! can be repeated in a smaller number of commands with this technique:

++.,.,?48

a3a3a3a3a3a3a3a3a3a3a3a3a3a3a3a3a3

Here, the same 4 commands are repeated 8 times, or rather in 8 cycles, so we iterate over 32 commands in just 7 inputs. These can be coupled for longer expressions.

Loops

An important rule to note is that, you cannot compound ? loops with ! loops, because ! loops are terminated to a zero after they are executed once. Here is an example iteration:

Step one
>>>,,>>!4. (loop detected, spaces = 4)
       ^
       i

Step two.... (print alpha detected)
>>>,,>>!0.
   ^
   i

Step six (! with no integer following, ignored)
>>>,,>>!0.
       ^
       i

I is meant to symbolize the current index of the input string the compiler is set to, and demonstrates the idea of terminating a loop. The same rule applies for ? loops, where each time the symbol is read, it subtracts one from the number 2 spaces after the ?, and then sets back the index pointer according to the number immedeatly following the ?.

Paths

In Farm, making an effecient, and readable program involves finding the best path to take to get to a particular character in the container, and then back to another character. For example, here is a program that prints c then r, then c again.

>>,<<>>>>>>>>>>>>>>>>>,<<<<<<<<<<<<<<<<<>>,

crc

One can notice that, for every print in the program, the pointer is moved forward to that letter, than moved all the way back to the 0 index. This method of finding the path to a symbol can be called the long hand. In a sense, every single character has a specific amount of forward and back commands to be reached, and the pointer is always set back to zero after each print.

In contrast, here is an example fo the short hand method:

>>,>>,>>>>>>>,,

cell

The short hand method always goes to the closest next letter after each print, without resetting it's pointer back to zero. Although this approach is faster, printing each symbol has a different set of commands to make the pointer reach it, which depends on the symbols you want to print before and after it.
